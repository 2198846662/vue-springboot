# 旅游平台 WorldThings 功能文档

## 一、项目概述

本项目为前后端分离的旅游景点展示平台，包含用户系统、景点管理、评论/回复/点赞、收藏等功能。

### 技术栈
- **后端**：Spring Boot 3.2 + JPA + MySQL + JWT
- **前端**：Vue 3 + Vite + TailwindCSS
- **数据库**：MySQL (库名: tourism)

---

## 二、数据表结构

### 1. user 用户表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键，自增 |
| username | VARCHAR(50) | 用户名，唯一，必填 |
| password | VARCHAR(255) | 密码(BCrypt加密)，必填 |
| email | VARCHAR(100) | 邮箱，唯一 |
| nickname | VARCHAR(50) | 昵称 |
| avatar | VARCHAR(255) | 头像URL |
| role | VARCHAR(20) | 角色：USER/ADMIN，默认USER |
| created_at | DATETIME | 创建时间 |

### 2. category 分类表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键，自增 |
| name | VARCHAR(50) | 分类名称，必填 |
| description | VARCHAR(255) | 分类描述 |
| icon | VARCHAR(50) | 图标名称 |
| sort_order | INT | 排序顺序，默认0 |
| created_at | DATETIME | 创建时间 |

### 3. scenery 景点表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键，自增 |
| name | VARCHAR(100) | 景点名称，必填 |
| description | TEXT | 景点描述 |
| cover_image | VARCHAR(500) | 封面图路径（支持本地上传后的`/uploads/xxx`或外链URL） |
| location | VARCHAR(200) | 地点 |
| ticket_price | DECIMAL(10,2) | 门票价格 |
| open_time | VARCHAR(50) | 开放时间 |
| latitude | DECIMAL(10,6) | 纬度（地图定位） |
| longitude | DECIMAL(10,6) | 经度（地图定位） |
| category_id | BIGINT | 所属分类ID，外键 |
| rating | DECIMAL(3,1) | 评分（前端使用0-5） |
| view_count | INT | 浏览次数，默认0 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

> 说明：地图组件使用 `latitude` + `longitude` 与景点关联，不依赖地址字符串解析。

### 4. comment 评论表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键，自增 |
| content | TEXT | 评论内容，必填 |
| scenery_id | BIGINT | 所属景点ID，外键，必填 |
| user_id | BIGINT | 评论用户ID，外键，必填 |
| rating | INT | 评分(1-5) |
| like_count | INT | 点赞数量，默认0 |
| created_at | DATETIME | 创建时间 |

### 5. favorite 收藏表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键，自增 |
| user_id | BIGINT | 用户ID，外键，必填 |
| scenery_id | BIGINT | 景点ID，外键，必填 |
| created_at | DATETIME | 收藏时间 |

**唯一约束**：user_id + scenery_id 唯一，防止重复收藏

### 6. comment_like 评论点赞表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键，自增 |
| comment_id | BIGINT | 评论ID，外键，必填 |
| user_id | BIGINT | 点赞用户ID，外键，必填 |
| created_at | DATETIME | 点赞时间 |

**唯一约束**：comment_id + user_id 唯一，防止同一用户重复点赞

### 7. 数据库迁移说明（增量）

如果是旧库升级（已存在 `scenery` 表），请执行：

## 四、API接口详细说明

### 1. 认证模块 /api/auth

#### POST /api/auth/register - 用户注册
- **认证**: 无需登录
- **请求体**:
```json
{
  "username": "string (必填)",
  "password": "string (必填)",
  "email": "string",
  "nickname": "string"
}
```
- **响应(成功)**:
```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "token": "jwt_token_string",
    "userId": 1,
    "username": "string",
    "nickname": "string",
    "role": "USER"
  }
}
```
- **响应(失败-用户名已存在)**:
```json
{
  "code": 400,
  "message": "用户名已存在",
  "data": null
}
```

#### POST /api/auth/login - 用户登录
- **认证**: 无需登录
- **请求体**:
```json
{
  "username": "string (必填)",
  "password": "string (必填)"
}
```
- **响应(成功)**:
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "jwt_token_string",
    "userId": 1,
    "username": "string",
    "nickname": "string",
    "role": "USER/ADMIN"
  }
}
```
- **响应(失败-密码错误)**:
```json
{
  "code": 401,
  "message": "用户名或密码错误",
  "data": null
}
```

### 2. 景点模块 /api/scenery

#### GET /api/scenery - 获取景点列表
- **认证**: 无需登录
- **查询参数**:
| 参数 | 类型 | 说明 |
|------|------|------|
| page | int | 页码(从0开始)，默认0 |
| size | int | 每页数量，默认10 |
| name | string | 按景点名称模糊搜索 |
| categoryId | Long | 按分类ID搜索 |
| category | string | 按分类名称搜索 |
| location | string | 按地点模糊搜索 |
| maxPrice | BigDecimal | 搜索低于此价格的景点 |
| sortBy | string | 排序字段，默认createdAt |
| sortDir | string | 排序方向，asc/desc，默认desc |

- **响应**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "content": [ /* Scenery对象数组 */ ],
    "totalElements": 100,
    "totalPages": 10,
    "number": 0,
    "size": 10
  }
}
```

#### GET /api/scenery/{id} - 获取景点详情
- **认证**: 无需登录
- **路径参数**: id - 景点ID
- **副作用**: 会自动将景点view_count +1
- **响应**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "scenery": { /* Scenery对象 */ },
    "category": { /* Category对象 */ }
  }
}
```

#### POST /api/scenery - 创建景点
- **认证**: 需要登录 + ADMIN角色
- **请求头**: `Authorization: Bearer <token>`
- **请求体**:
```json
{
  "name": "string (必填)",
  "description": "string",
  "coverImage": "string",
  "location": "string",
  "ticketPrice": "BigDecimal",
  "openTime": "string",
  "latitude": "BigDecimal",
  "longitude": "BigDecimal",
  "categoryId": "Long",
  "rating": "BigDecimal"
}
```
- **响应(成功)**:
```json
{
  "code": 200,
  "message": "创建成功",
  "data": { /* 新创建的Scenery对象 */ }
}
```
- **响应(失败-无权限)**:
```json
{
  "code": 403,
  "message": "需要管理员权限",
  "data": null
}
```

#### PUT /api/scenery/{id} - 更新景点
- **认证**: 需要登录 + ADMIN角色
- **请求头**: `Authorization: Bearer <token>`
- **请求体**: 同POST
- **响应**: 同POST

#### DELETE /api/scenery/{id} - 删除景点
- **认证**: 需要登录 + ADMIN角色
- **请求头**: `Authorization: Bearer <token>`
- **响应**:
```json
{
  "code": 200,
  "message": "删除成功",
  "data": null
}
```

## 五、用户交互流程

### 1. 注册登录流程
```
1. 用户打开页面 → 点击登录/注册
2. 输入用户名密码 → 调用 POST /api/auth/register 或 POST /api/auth/login
3. 后端验证并返回 JWT token
4. 前端存储 token 到 localStorage.currentUser
5. 后续请求通过 Authorization 头携带 token
```

### 2. 浏览景点流程
```
1. 首页加载 → 调用 GET /api/scenery 获取景点列表
2. 点击景点卡片 → 调用 GET /api/scenery/{id} 获取详情
3. 自动增加 view_count
```

### 3. 发表评论流程
```
1. 用户登录后进入景点详情页
2. 在评论框输入内容 → 点击发布
3. 调用 POST /api/scenery/{sceneryId}/comments
   (携带 Authorization 头)
4. 后端从 token 获取 userId 创建评论
5. 前端将新评论添加到列表顶部
```

### 4. 收藏景点流程
```
1. 用户登录后进入景点详情页
2. 点击收藏按钮
3. 调用 POST /api/scenery/{sceneryId}/favorite
4. 后端切换收藏状态
5. 前端更新按钮状态
```

### 5. 删除评论流程
```
1. 用户在评论列表看到自己的评论
2. 点击删除按钮
3. 调用 DELETE /api/scenery/{sceneryId}/comments/{id}
4. 后端验证：是否为评论所有者 或 管理员
5. 前端从列表移除该评论
```

### 6. 点赞评论流程
```
1. 用户登录后在评论区点击点赞
2. 调用 POST /api/scenery/{sceneryId}/comments/{id}/like
3. 后端根据用户与评论关系执行点赞或取消点赞
4. 返回最新 likeCount 与当前 liked 状态
5. 前端实时更新点赞图标与数量
```

### 7. 我的收藏列表流程
```
1. 用户登录后进入“我的收藏”页面
2. 调用 GET /api/favorites
3. 后端根据 token 返回该用户收藏景点列表
4. 前端渲染收藏卡片并支持跳转景点详情
```

## 六、安全机制

### 1. JWT Token认证
- 生成: 登录/注册时生成，有效期24小时
- 结构: 包含 userId 和 username
- 验证: 通过 LoginInterceptor 拦截器验证
- 密钥: `tourism-platform-secret-key-2024-very-long-for-hs256`

### 2. 密码加密
- 算法: BCrypt
- 存储: 数据库中存储加密后的哈希值

### 3. 权限控制
- 用户操作: 通过 @RequireAdmin 注解 + AdminAspect 切面
- 删除校验: Controller层验证是否为所有者或管理员

---

## 七、已知限制

1. **评论不支持修改**：当前仅支持创建、删除、回复与点赞
2. **用户没有独立管理接口**：无法查看/编辑其他用户信息
3. **分类搜索景点有限制**：前端传入category参数时按名称搜索，不是ID

## 八、测试账号

由于密码使用BCrypt加密，需要通过注册接口创建账号：

1. **注册普通用户**: POST /api/auth/register
   ```json
   {
     "username": "testuser",
     "password": "123456"
   }
   ```

2. **注册管理员**: 先注册普通用户，然后手动更新数据库：
   ```sql
   UPDATE user SET role = 'ADMIN' WHERE username = 'testuser';
   ```
